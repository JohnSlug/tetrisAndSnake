Для управления процессом игры Вы можете использовать следующие клавиши\+:


\begin{DoxyItemize}
\item {\itshape {\bfseries{s}}} -\/ начало игры, инициализация игрового и информационного полей;
\item {\itshape {\bfseries{Стрелка влево}}} -\/ движение фигуры по горизонтали влево;
\item {\itshape {\bfseries{Стрелка вправо}}} -\/ движение фигуры по горизонтали вправо;
\item {\itshape {\bfseries{Стрелка вниз}}} -\/ движение фигуры по вертикали до конца игрового поля;
\item {\itshape {\bfseries{Пробел}}} -\/ поворот фигуры на 90 градусов;
\item {\itshape {\bfseries{p}}} -\/ постановка игры на паузу/возобновление игры после паузы;
\item {\itshape {\bfseries{q}}} -\/ выход из игры.
\end{DoxyItemize}

В данной игре присутствует информационное поле, которое находится {\itshape справа} от игрового. На нем можно отслеживать следующую информацию\+:


\begin{DoxyItemize}
\item текущее количество очков;
\item текущий рекорд;
\item уровень;
\item скорость;
\item тип следующей фигуры.
\end{DoxyItemize}

За уничтожение линий игрок получает количество очков, соответствующее количеству уничтоженных линий\+:


\begin{DoxyItemize}
\item {\bfseries{1 линия}} -\/ +100 очков;
\item {\bfseries{2 линии}} -\/ +300 очков;
\item {\bfseries{3 линии}} -\/ +700 очков;
\item {\bfseries{4 линии}} -\/ +1500 очков;
\end{DoxyItemize}

В игре есть {\itshape {\bfseries{10 уровней}}}. Каждый уровень прямо влияет на ускорение падения фигуры. Уровень повышается каждые {\bfseries{600 очков}}. {\bfseries{Максимальное количество уровней -\/ 10}}.

При достижении фигурой верхнего края игрового поля {\bfseries{игра завершается}}.\hypertarget{md__read_me_autotoc_md3}{}\doxysection{\texorpdfstring{Логическая организация игры}{Логическая организация игры}}\label{md__read_me_autotoc_md3}
Логическая составляющая игры формализована в виде такой математической абстракции, как {\bfseries{конечный автомат}}. Идея заключается в том, что программа находится лишь в одном состоянии в каждый момент времени. Состояние меняется в зависимости от ввода пользователя и от хода самой игры.\hypertarget{md__read_me_autotoc_md4}{}\doxysubsection{\texorpdfstring{Схема реализации конечного автомата в данной игре\+:}{Схема реализации конечного автомата в данной игре\+:}}\label{md__read_me_autotoc_md4}


Согласно данной схеме, игра может находиться в следующих состояниях\+:


\begin{DoxyItemize}
\item {\bfseries{GAME\+\_\+\+START}}. Состояние, в котором происходит инициализация игры, включая создание игрового поля, генерацию первой фигуры и установку начальных значений для счета, уровня и скорости.
\item {\bfseries{GAME\+\_\+\+SPAWN}}. Состояние, в котором генерируется новая фигура и помещается на игровое поле.
\item {\bfseries{GAME\+\_\+\+MOVING}}. Состояние, в котором фигура движется по игровому полю в соответствии с нажатыми клавишами и правилами игры.
\item {\bfseries{GAME\+\_\+\+ATTACHING}}. Состояние, в котором проверяется, не столкнулась ли фигура с другими фигурами или границами игрового поля, и, если да, то фигура фиксируется на игровом поле, а также проверяется, не было ли уничтожено полных линий.
\item {\bfseries{GAME\+\_\+\+OVER}}. Состояние, в котором игрок видит сообщение об окончании игры и может нажать кнопку "{}q"{}, чтобы выйти из игры.
\item {\bfseries{GAME\+\_\+\+PAUSED}}. Постановка игры на паузу, предыдущее состояние сохраняется. При повторном нажатии возобновление игрового процесса.
\item {\bfseries{GAME\+\_\+\+EXIT}}. Выход из игры. Программа переходит в данное состояние при решении пользователя завершить игровой процесс.

Когда змейка сталкивается с «яблоком», ее длина увеличивается на один.

В игре есть {\itshape {\bfseries{10 уровней}}}. Каждый раз, когда игрок набирает 5 очков, уровень увеличивается на 1. Повышение уровня увеличивает скорость движения змейки. Максимальное количество уровней — 10.

Если змейка сталкивается с границей поля или сама с собой, {\bfseries{игра заканчивается поражением игрока}}.
